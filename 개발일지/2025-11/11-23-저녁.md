# 개발일지 - 2025년 11월 23일 저녁

## 완료한 작업

### 유니크 조회수 시스템 구현
- PostView 모델 추가 (사용자당 1회만 조회수 증가)
- Transaction으로 view 기록 생성 및 카운트 증가
- 중복 조회 방지 (postId_userId unique constraint)

### 커뮤니티 UI 대폭 개선
- 카테고리 구분선 추가 (border-t)
- 커뮤니티 정보 섹션 2열 레이아웃으로 재구성
  - 왼쪽: 동아리 정보 (소유자, 통계)
  - 오른쪽: 나의 활동 (역할, 받은 추천, 게시글, 댓글)
- 설정 아이콘 추가 (유저 프로필 옆에 배치)

### 가입 프로세스 구현
- 닉네임 실시간 중복 검사 (500ms debounce)
- 닉네임과 기존 유저네임 충돌 방지
- Byte 계산 시스템 (한글 3바이트, 20바이트 제한)
- 시각적 피드백 (녹색 체크, 빨간 X, 로딩)
- 가입 페이지 UI 구현
  - 프로필 이미지
  - 별명
  - 공개 설정
  - 가입 질문

### 설정 페이지 구현
- 별명 변경 기능 (NicknameInput 컴포넌트 재사용)
- 게시판 차단 기능 UI (체크박스)
- 현재 별명 표시
- `/communities/[id]/settings` 경로

### 탈퇴한 부원 표시 기능
- 커뮤니티 멤버 배열로 멤버십 확인
- 탈퇴한 부원 처리
  - 기본 프로필 사진 + "탈퇴한 부원" 텍스트 (회색)
  - 게시글 목록, 게시글 상세, 댓글에 모두 적용
  - 게시글/댓글 내용은 그대로 유지
- 탈퇴 시 닉네임 재사용 가능 (CommunityMember 삭제됨)
